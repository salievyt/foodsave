# FoodSave — Спецификация дизайна приложения

---

## 1. Общее описание приложения

**Название:** FoodSave  
**Тип:** Мобильное приложение (iOS/Android)  
**Слоган:** "Сохрани продукты — сэкономь и природу"  
**Целевая аудитория:** 25-45 лет, заботятся о здоровье, экологии и бюджете

### Суть приложения
Приложение помогает пользователям отслеживать продукты в холодильнике, чтобы не забыть их вовремя использовать, находить рецепты на основе имеющихся продуктов и минимизировать пищевые отходы.

## 3. Экраны приложения

### 3.1 Splash Screen (Загрузка)

**Описание:** Экран при запуске приложения  
**Элементы:**
- Логотип приложения (центр)
- Название "FoodSave" под логотипом
- Анимация появления (fade in)
- Автоматический переход через 2-3 секунды

---

### 3.2 Onboarding (Онбординг)

**Описание:** 3-4 экрана при первом запуске  
**Экраны:**

1. **Главный экран**
   - Иллюстрация холодильника с продуктами
   - Заголовок: "Отслеживай продукты"
   - Описание: "Добавь продукты в холодильник и следи за сроками годности"

2. **Рецепты**
   - Иллюстрация готовки
   - Заголовок: "Находи рецепты"
   - Описание: "Приложение подберёт рецепты из твоих продуктов"

3. **Статистика**
   - Иллюстрация графика
   - Заголовок: "Экономь и помогай природе"
   - Описание: "Смотри сколько продуктов ты спас от мусорки"

4. **Завершение**
   - Кнопка "Начать"
   - Переход на регистрацию/вход

---

### 3.3 Авторизация

#### 3.3.1 Login (Вход)

**Элементы:**
- Логотип сверху
- Поле "Email" (текстовое поле)
- Поле "Пароль" (пароль, скрыт)
- Кнопка "Войти" (Primary цвет)
- Ссылка "Забыли пароль?"
- Ссылка "Нет аккаунта? Регистрация"
- Кнопки входа через Google, Apple (опционально)

#### 3.3.2 Register (Регистрация)

**Элементы:**
- Поле "Имя пользователя"
- Поле "Email"
- Поле "Пароль"
- Поле "Подтверждение пароля"
- Чекбокс "Согласен с условиями"
- Кнопка "Зарегистрироваться"
- Ссылка "Есть аккаунт? Вход"

---

### 3.4 Главный экран (Bottom Navigation)

**Описание:** 5 вкладок в нижней навигации  
**Вкладки:**

1. **Главная (Home)** — домик
2. **Холодильник (Fridge)** — холодильник
3. **Сканер (Scanner)** — сканер/камера
4. **Рецепты (Recipes)** — книга рецептов
5. **Профиль (Profile)** — человек

---

### 3.5 Home (Главная)

**Элементы:**
- Приветствие "Привет, [Имя]!" + дата
- Карточка "Скоро истекает" — список из 2-3 продуктов, которые нужно срочно использовать
- Кнопка "Что приготовить?" — переход к рецептам
- Статистика за неделю:
  - "Спасено продуктов: X"
  - "Выброшено: Y"
- Ближайший праздник/событие (опционально)

---

### 3.6 Fridge (Холодильник)

**Описание:** Список всех продуктов пользователя  
**Элементы:**

- Заголовок "Мой холодильник"
- Поиск продуктов
- Фильтры по категориям (Все, Овощи, Фрукты, Молочное, Мясо, Напитки и т.д.)
- Сортировка (По дате, По названию, По статусу)
- Список продуктов:
  - Название продукта
  - Количество + единица измерения
  - Дата добавления
  - Срок годности с индикатором цвета (свежий/скоро/срочно/просрочен)
  - Кнопка редактирования/удаления
- FAB "Добавить продукт"

#### Добавление продукта (модалка/экран)

- Поле "Название" (с автодополнением)
- Категория (выбор из списка)
- Количество + единица (шт, кг, г, л, мл)
- Дата добавления (по умолчанию сегодня)
- Срок годности (выбор даты)
- Заметки (опционально)
- Кнопка "Сохранить"

---

### 3.7 Scanner (Сканер)

**Описание:** Сканирование штрих-кодов и добавление продуктов  
**Элементы:**
- Камера с рамкой сканирования
- Вспышка (вкл/выкл)
- Ручной ввод (ссылка)
- При успешном сканировании:
  - Название продукта (подтягивается из базы)
  - Предложенные категории
  - Поля: количество, срок годности
  - Кнопка "Добавить в холодильник"

---

### 3.8 Recipes (Рецепты)

**Описание:** Список рецептов на основе продуктов в холодильнике  
**Элементы:**

- Заголовок "Что приготовить?"
- Описание: "Рецепты отсортированы по совпадению с продуктами в холодильнике"
- Секция "Срочно использовать" — чипсы с продуктами, которые скоро истекут
- Список рецептов (карточки):
  - Изображение/эмодзи блюда
  - Название рецепта
  - Время приготовления
  - Сложность (Легко/Средне/Сложно)
  - Процент совпадения с продуктами (X% совпадение)
  - Кнопка "Избранное"
- Пагинация (бесконечный скролл)

#### Детали рецепта

- Большое изображение/эмодзи
- Название
- Время, сложность, порции
- Теги (завтрак, обед, ужин, десерт)
- Ингредиенты (список)
- Пошаговая инструкция
- Кнопка "Добавить недостающие в холодильник"
- Кнопка "Поделиться"

---

### 3.9 Profile (Профиль)

**Элементы:**

- Аватар (фото или дефолтная иконка)
- Имя пользователя
- Email
- Статус премиума (если есть)
- Статистика:
  - Всего спасено продуктов
  - Выброшено продуктов
  - Дней в приложении
- Меню:
  - Уведомления (настройка)
  - Диетические предпочтения
  - Аллергии
  - Тёмная тема (вкл/выкл)
  - Язык
  - О приложении
  - Помощь/FAQ
  - Выход

---

### 3.10 Statistics (Статистика)

**Описание:** Подробная статистика использования  
**Элементы:**
- Заголовок "Твоя статистика"
- График "Спасено vs Выброшено" (за неделю/месяц/год)
- Общие цифры:
  - Всего спасено
  - Всего выброшено
  - Процент спасённых
- Рейтинг категорий (какие продукты чаще спасаете)
- Достижения/бейджи (опционально)
- Поделиться статистикой

### 3.11 Support Chat (Поддержка)

**Описание:** Чат с поддержкой  
**Элементы:**
- Заголовок "Поддержка"

## 4. Компоненты UI (общие)

### Навигация
- Bottom Navigation Bar (5 иконок)
- Назад кнопка (стрелка влево)
- Закрыть (крестик)

### Кнопки
- Primary: `#E53935` фон, белый текст, скругление 12px
- Secondary: прозрачный фон, `#E53935` текст и бордер
- Text Button: без фона, цветной текст
- FAB: круглая, Primary цвет, иконка +

### Поля ввода
- Высота 52px
- Скругление 12px
- Серый фон `#F2F2F7`
- Фокус: Primary цвет бордера (2px)
- Плейсхолдер: Text Secondary

### Карточки
- Белый фон (Surface)
- Скругление 24px
- Тень: `rgba(0,0,0,0.06)`, blur 15, offset (0, 6)

### Списки
- Разделитель между элементами
- Chevron (стрелочка) справа для кликабельных

### Снекбар (уведомления)
- Внизу экрана
- Полупрозрачный чёрный фон
- Белый текст
- Кнопка действия (опционально)

## 6. Анимации и переходы

- Переход между экранами: fade (300ms)
- Загрузка: skeleton (анимация волн)
- Кнопки: scale при нажатии (0.95)
- Добавление в избранное: heart fill анимация
- Pull-to-refresh: стандартный индикатор
