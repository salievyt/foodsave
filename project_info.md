# FoodSave — Project Info

## 1. Summary
FoodSave is a mobile app that helps users track food items, reduce waste, and save money. It maintains a digital fridge, highlights items nearing expiration, and suggests recipes based on what is available. The project includes a Flutter client and a Django REST backend with real‑time support chat via WebSockets.

## 2. Goals
- Reduce food waste by tracking expiry dates.
- Provide quick add flows (manual add and receipt scanning).
- Offer recipe suggestions from current inventory.
- Provide a support channel inside the app.

## 3. Target Users
- People who want to manage groceries efficiently.
- Users who often forget items in the fridge.
- Anyone looking to reduce food waste.

## 4. Core Features
- Authentication (JWT).
- Digital fridge inventory with status tracking.
- Receipt scanning endpoint (mock OCR, currently persists scanned items).
- Recipe browsing and matching.
- In‑app support chat (WebSockets).
- Local caching for faster UI.
- Onboarding flow and guest access.

## 5. Tech Stack
### Frontend
- Flutter (Riverpod, AutoRoute, Dio, SharedPreferences)
- WebSocket channel for chat
- Local notifications for expiring items

### Backend
- Django + DRF
- Django Channels + Daphne
- JWT auth (SimpleJWT)
- SQLite for development

## 6. Architecture
### Frontend (Feature-first)
- `lib/core` — routing, theming, shared services
- `lib/features` — auth, fridge, recipes, profile, support, statistics, onboarding

### Backend
- `users` — user model, auth endpoints
- `fridge` — inventory + receipt scan
- `recipes` — recipe data and API
- `notifications` — support chat and notifications

## 7. Environments & Config
### Flutter
Uses `.env`:
- `API_BASE_URL`
- `WS_BASE_URL`

### Django
Uses `.env`:
- `DJANGO_SECRET_KEY`
- `DEBUG`
- `ALLOWED_HOSTS`
- `CORS_ALLOW_ALL_ORIGINS`

## 8. Data Seeds
Backend fixtures:
- `food_save_backend/fixtures/seed.json`
Load with:
```
python manage.py loaddata fixtures/seed.json
```
Create demo user:
```
python manage.py seed_demo
```

## 9. Guest Mode
Guest login creates a unique account with autogenerated credentials and stores JWT tokens locally. This isolates guest data in the database.

## 10. Known Limitations
- Recipe “AI” is currently a keyword match, not a true AI model.
- Receipt OCR is mocked, not real OCR.
- Minimal automated tests.

## 11. Runbook
### Backend
```
cd food_save_backend
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver 0.0.0.0:8003
```

### Frontend
```
cd food_save
cp .env.example .env
flutter pub get
flutter run
```
