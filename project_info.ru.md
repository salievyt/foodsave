# FoodSave — Описание проекта

## 1. Кратко о продукте
FoodSave — мобильное приложение для учета продуктов, снижения пищевых отходов и экономии. Оно ведет цифровой холодильник, выделяет товары с близким сроком годности и предлагает рецепты на основе доступных ингредиентов. Проект состоит из Flutter‑клиента и Django‑backend с поддержкой чата через WebSockets.

## 2. Цели
- Снижение пищевых отходов через контроль сроков годности.
- Быстрое добавление продуктов (вручную и по чеку).
- Подбор рецептов по текущему инвентарю.
- Встроенная поддержка в приложении.

## 3. Целевая аудитория
- Пользователи, которые хотят системно управлять продуктами.
- Те, кто часто забывает про еду в холодильнике.
- Люди, которые хотят меньше выбрасывать и экономить.

## 4. Основные функции
- Авторизация (JWT).
- Цифровой холодильник и статусы продуктов.
- Сканирование чека (mock OCR, с сохранением результатов).
- Рецепты и совпадение по ингредиентам.
- Чат поддержки (WebSockets).
- Локальный кеш для быстрого UI.
- Онбординг и гостевой вход.

## 5. Стек
### Flutter
- Flutter (Riverpod, AutoRoute, Dio, SharedPreferences)
- WebSocket для чата
- Локальные уведомления о сроках годности

### Backend
- Django + DRF
- Django Channels + Daphne
- JWT (SimpleJWT)
- SQLite для разработки

## 6. Архитектура
### Frontend (Feature‑first)
- `lib/core` — роутинг, темы, общие сервисы
- `lib/features` — auth, fridge, recipes, profile, support, statistics, onboarding

### Backend
- `users` — модель пользователя и auth
- `fridge` — инвентарь + скан чека
- `recipes` — рецепты и API
- `notifications` — чат поддержки и уведомления

## 8. Сиды данных
Фикстуры:
- `food_save_backend/fixtures/seed.json`

Загрузка:
```
python manage.py loaddata fixtures/seed.json
```

Создание тестового пользователя:
```
python manage.py seed_demo
```

## 9. Гостевой режим
Гостевой вход создает уникального пользователя с автоматически сгенерированными данными и сохраняет JWT‑токены локально. Данные гостей не пересекаются.

## 10. Текущие ограничения
- «AI» рецептов — это совпадение по ключевым словам, не полноценная модель.
- OCR чеков пока моковый.
- Минимальное покрытие тестами.


## 11. Возможные улучшения
- Реальный OCR вместо мокового.
- Подключение AI/ML для рецептов.
- Расширение тестов.
- Аналитика и метрики поведения пользователей.
